[State Variant Rules]
When a character or entity undergoes a SIGNIFICANT VISUAL CHANGE across the story that adds or removes major visual elements attached to its body/surface, you MUST create a SEPARATE character entry for each visually distinct state.

**Naming Convention:** "BaseName (State Description)"
- The base/default state should also have a state qualifier: "Truck (Empty)", NOT just "Truck"
- Each variant must have its own unique `appearance` describing that specific visual state

**WHEN TO CREATE A STATE VARIANT — only when major visual elements are ADDED TO or REMOVED FROM the entity:**

Example 1 — Vehicle with cargo changes:
- "Truck (Empty)" → plain truck with empty flatbed
- "Truck (Loaded with Logs)" → same truck but now with timber logs stacked high on the flatbed, secured with straps
- "Truck (Towed by Rescue Vehicle)" → same truck now connected by heavy chains to a tow truck, being pulled

Example 2 — Person with major outfit/equipment change:
- "Knight (Casual Clothes)" → young man in linen tunic and leather boots
- "Knight (Full Battle Armor)" → same man now wearing full plate armor, helmet, shield on back, sword at hip

Example 3 — Animal with transformation:
- "Caterpillar" → a green caterpillar with yellow stripes on a leaf
- "Butterfly" → a monarch butterfly with orange-and-black wings (transformed from the caterpillar)

Example 4 — Object with attached elements:
- "Fishing Boat (Clean)" → small wooden fishing boat with nets
- "Fishing Boat (Decorated for Festival)" → same boat now covered in colorful flags, lanterns, and flower garlands

**WHEN NOT TO CREATE A STATE VARIANT — these are handled by video/shot prompts, NOT by creating new characters:**

- Movement/position changes: truck driving vs truck parked — same entity, just moving
- Orientation changes: truck standing upright vs truck flipped on its side — same entity, just rotated
- Facial expressions: character smiling vs character crying — same face, just expression change
- Temporary actions: character raising hand vs lowering hand — same character, momentary pose
- Camera angle: close-up vs wide shot of same character — same character, different framing
- Lighting: character in daylight vs character at night — same character, different environment (handled by Scene/Background)
- Damage without added elements: a cracked window, a dented car — still the same object visually

**Key test:** Would you need to DRAW ADDITIONAL OBJECTS on the reference image to accurately depict this state? If YES → create a variant. If NO → don't.
